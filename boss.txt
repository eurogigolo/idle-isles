Grid Boss Encounter - Current Implementation Details
====================================================

Last updated
------------
- Date: 2026-06-25.
- Purpose: Track the current boss battle implementation and the near-term design direction for sector bosses and equipment impact.

High-level boss direction
-------------------------
- Idle Galactica is structured around sectors as progression chapters.
- Each sector should eventually have its own capstone boss.
- Each boss should teach or test the mechanics, resources, and ship-building priorities of its sector.
- Boss fights should remain skill-based arcade encounters, but equipped ship modules should noticeably improve the player's odds.
- Gear should make boss fights smoother and more forgiving, not automatic.
- The preferred long-term shape is:
  - Sector missions introduce resources, threats, and upgrade goals.
  - Production creates or improves the modules that help with that sector's boss.
  - The sector boss acts as the capstone test.
  - Future rewards can make boss kills matter economically once the fight loop feels good.

Sector boss philosophy
----------------------
- Orbital Dock:
  - Role: starter boss and tutorial capstone.
  - Current boss: Rift Warden.
  - Focus: movement, dodging, charge shots, row telegraphs, and panel lockdowns.
- Inner Belt:
  - Role: first real progression boss.
  - Current boss: Belt Breaker.
  - Focus: heavier pressure, multi-row volleys, stronger phases, and module preparation.
- Future sectors:
  - Each sector should get a different boss identity and attack language.
  - Examples:
    - Nebula sector boss: visibility distortion, slow fields, energy attacks.
    - Wreck sector boss: drones, mines, scrap hazards, disable effects.
    - Deep space boss: phase shifts, gravity pulls, unstable panels.

Equipment impact philosophy
---------------------------
- Gear should provide minor but meaningful boss-fight modifiers.
- The best balance target is "noticeably better" rather than "required to win."
- Hardpoints should improve offense.
- Shield arrays and hull plating should improve survival.
- Engines should improve movement feel.
- Sensor suites should improve warning/telegraph windows.
- Auxiliary modules should improve utility effects such as charge damage or panel lockdown delay.
- The current modifier system is intentionally capped so later gear tiers can be added without breaking early bosses.

Adding a new sector boss
------------------------
- Add or confirm the sector in src/game.ts.
- Add a SectorBoss entry in src/bosses.ts.
- Define:
  - boss id.
  - display name.
  - sector id.
  - base boss HP.
  - base player boss-fight hull.
  - phase numbers.
  - attack pattern rows.
  - rift targeting behavior.
  - UI mechanics labels.
  - victory/defeat/active text.
- Reuse GridBossBattle unless the boss needs a new projectile, telegraph, or arena rule.
- If a boss needs a new attack family, add the smallest reusable attack concept to GridBossBattle instead of hardcoding sector-specific branches.
- Update this file with exact numbers after implementation.
- Run:
  - npm.cmd run lint.
  - npm.cmd run content:check.
  - npm.cmd run build.

Source files
------------
- Sector boss data and gear modifiers: src/bosses.ts
- Frontend battle logic: src/GridBossBattle.tsx
- Fight button / Boss tab flow: src/App.tsx
- Battle styling: src/App.css
- On-chain entry point: contracts/IdleGalactica.sol

Core model
----------
- Bosses are selected by current sector.
- Current sectors with bosses:
  - orbitalDock -> Rift Warden.
  - innerBelt -> Belt Breaker.
- The battle engine is shared.
- Boss identity, HP, phase numbers, attack row patterns, text, and mechanics are data-driven through SectorBoss.
- Bosses can define their own visual theme:
  - rift.
  - belt.
- Bosses can define their Phase 3 special attack kind:
  - rift.
  - crush.
- Bosses can define their volley projectile kind:
  - bolt.
  - shrapnel.
- Equipped ship modules create BossBattleModifiers before the battle starts.
- The modifiers are snapshotted from the current game state when GridBossBattle mounts.
- The contract still charges the generic boss encounter entry cost; the frontend chooses the sector boss.

On-chain entry flow
-------------------
- Chain Mode is required for the current boss entry flow.
- A chain profile is required.
- The configured game contract must expose BOSS_ENCOUNTER_COST().
- Current Solidity cost: BOSS_ENCOUNTER_COST = 1.
- Current frontend fallback cost: 1 Credit.
- Credit item id: CREDITS = 1.
- startBossEncounter() behavior:
  - Requires an existing profile.
  - Settles any active mission first via _settle(player).
  - Reverts with CargoLow() if the player has fewer than 1 Credit.
  - Burns exactly 1 Credit.
  - Emits BossEncounterStarted(player, 1).
- Frontend transaction path:
  - MOSS wallet mode calls writeMossStartBossEncounter().
  - Injected wallet mode calls writeStartBossEncounter().
  - After the write call resolves, App.tsx sets bossBattleId to Date.now().
  - GridBossBattle mounts with the current sector boss and current equipment modifiers.

Grid layout
-----------
- Total grid: 3 rows x 6 columns.
- Rows: 0, 1, 2 from top to bottom.
- Columns: 0, 1, 2, 3, 4, 5 from left to right.
- Player zone: left 3 columns, columns 0 through 2.
- Boss zone: right 3 columns, columns 3 through 5.
- Boss local grid: 3 rows x 3 columns.
- Boss local columns: 0, 1, 2.
- Boss local rows: 0, 1, 2.
- Boss global column = BOSS_ZONE_START + bossCoreCol.
- BOSS_ZONE_START = 3.
- Boss occupies one boss-zone panel at a time, not the full 3x3 area.
- Player panel count: ROWS * PLAYER_COLUMNS = 3 * 3 = 9.
- Maximum disabled player panels: 8.
- At least 1 player panel is always left usable.

Base player controls
--------------------
- Keyboard movement:
  - ArrowUp or W.
  - ArrowDown or S.
  - ArrowLeft or A.
  - ArrowRight or D.
- Keyboard fire:
  - Space.
  - Enter.
- Touch / button controls:
  - Up.
  - Left.
  - Fire.
  - Right.
  - Down.
- Base movement cooldown: 95 ms.
- Movement cooldown after modules = 95 ms * moveCooldownMultiplier.
- Player movement is clamped to columns 0 through 2 and rows 0 through 2.
- Player cannot move into a disabled panel.
- Releasing Fire resolves the current normal or charged shot.

Player firing
-------------
- Base normal shot cooldown: 220 ms.
- Base charge shot cooldown: 360 ms.
- Base charge minimum: 360 ms.
- Base full charge: 950 ms.
- Charge timing after modules:
  - chargeMinMs = round(360 * chargeTimeMultiplier).
  - chargeMaxMs = round(950 * chargeTimeMultiplier).
- Normal projectile:
  - kind: bolt.
  - owner: player.
  - start x: playerCol + 0.72.
  - speed: 0.011 grid columns per ms to the right.
  - damage: 1 + playerDamageBonus.
- Charge projectile:
  - kind: charge.
  - owner: player.
  - start x: playerCol + 0.72.
  - speed: 0.013 grid columns per ms to the right.
  - minimum charge damage: 3 + chargeDamageBonus.
  - full charge damage: 5 + chargeDamageBonus.
- Player shots increment shotsFired.
- Charge shots increment chargeShotsFired.
- Successful player hits increment shotsHit and damageDealt.
- Successful charge hits increment chargeShotsHit.

Equipment modifiers
-------------------
- Boss modifiers are computed in getBossBattleModifiers(game).
- Source stats are getShipStats(game), which sums equipped module stats.
- Module stat names:
  - damage.
  - accuracy.
  - shielding.
  - armor.
  - hull.
  - speed.
  - utility.
- playerDamageBonus:
  - min(2, floor(damage / 8)).
- chargeDamageBonus:
  - min(2, floor(damage / 12) + floor(utility / 3)).
- playerMaxHpBonus:
  - min(3, ceil(hull / 15)).
- incomingDamageReduction:
  - min(2, floor((shielding + armor) / 8)).
- telegraphBonusMs:
  - min(220, accuracy * 25 + utility * 20).
- panelLockdownDelayMs:
  - min(650, utility * 140).
- moveCooldownMultiplier:
  - max(0.72, 1 - speed * 0.055).
- chargeTimeMultiplier:
  - max(0.78, 1 - accuracy * 0.025).
- Incoming boss damage after modules:
  - max(1, projectileDamage - incomingDamageReduction).

Module impact examples
----------------------
- Light Turret:
  - stats: damage 8, accuracy 4.
  - boss effects: +1 normal shot damage, faster charge timing, longer telegraphs.
- Missile Launcher:
  - stats: damage 16, accuracy 2.
  - boss effects: +2 normal shot damage, +1 charge damage, slightly faster charge timing, longer telegraphs.
- Basic Shield Generator:
  - stats: shielding 5.
  - boss effects: contributes toward incoming damage reduction.
- Reinforced Hull Plating:
  - stats: armor 5, hull 18.
  - boss effects: contributes toward damage reduction and +2 boss-fight hull.
- Basic Thrusters:
  - stats: speed 3, accuracy 1.
  - boss effects: about 16% faster movement, slightly faster charge timing, longer telegraphs.
- Survey Scanner:
  - stats: accuracy 2, utility 2.
  - boss effects: faster charge timing, longer telegraphs, +280 ms panel lockdown delay.
- Drone Bay:
  - stats: utility 3, shielding 1.
  - boss effects: +1 charge damage, +420 ms panel lockdown delay, longer telegraphs.

Rift Warden attack pattern
--------------------------
- Wide row sequence:
  - 0, 2, 1, 0, 2, 1, then repeats.
- Volley row sequence:
  - [0, 2]
  - [0, 1]
  - [1, 2]
  - [2, 0]
  - then repeats by special index.
- Phase 3 rift cadence:
  - Every 3rd special when specialIndex % 3 == 2.
- Rift target:
  - Player row at telegraph creation time.
  - It does not keep homing after telegraph creation.
- Phase 3 extra normal bolt:
  - Enabled.
  - Every other normal shot fires a second bolt.
  - Second bolt row = (bossCoreRow + 1) % 3.

Boss movement
-------------
- Movement begins only after countdown completes.
- On each movement tick, the boss chooses one random valid cardinal step.
- Allowed directions:
  - Up by 1 row.
  - Down by 1 row.
  - Left by 1 local column.
  - Right by 1 local column.
- No diagonal boss movement.
- Boss cannot leave its local 3x3 grid.

Panel lockdown
--------------
- Panel lockdown is separate from the boss special cooldown.
- It randomly disables one currently usable player-zone panel.
- It can target any open panel from the player's 3x3 area.
- Disabled panels are permanent for the current fight.
- Disabled panels block player movement.
- Max disabled panels: 8 of 9.
- If the newly disabled panel is the player's current panel:
  - The player is moved to the nearest open panel.
  - Distance metric: Manhattan distance.
  - Ties use the existing row-major open panel order.
- A disabled panel increments panelsDisabled.
- A disabled panel creates theme-specific particles and light screen shake:
  - rift theme: rift particles.
  - belt theme: shrapnel particles.

Projectile movement and hitboxes
--------------------------------
- Main animation loop uses requestAnimationFrame.
- Per-frame delta:
  - deltaMs = min(40, now - previous).
- Projectile update:
  - next x = current x + projectile.speed * deltaMs.
- Projectile row never changes after creation.
- Boss attacks are straight-line projectiles, not homing projectiles.
- Collision is based on ship/projectile overlap, not panel occupancy alone.
- Player ship hitbox:
  - centerX = playerCol + 0.5.
  - centerY = playerRow + 0.5.
  - halfWidth = 0.18.
  - halfHeight = 0.14.
- Boss hitbox:
  - centerX = 3 + bossCoreCol + 0.5.
  - centerY = bossCoreRow + 0.5.
  - halfWidth = 0.22.
  - halfHeight = 0.20.
- Bolt hitbox:
  - halfWidth = 0.07.
  - halfHeight = 0.035.
- Shrapnel hitbox:
  - halfWidth = 0.09.
  - halfHeight = 0.06.
- Charge shot hitbox:
  - halfWidth = 0.16.
  - halfHeight = 0.06.
- Wide projectile hitbox:
  - halfWidth = 0.46.
  - halfHeight = 0.22.
- Rift projectile hitbox:
  - halfWidth = 0.20.
  - halfHeight = 0.12.
- Crush beam hitbox:
  - halfWidth = 0.32.
  - halfHeight = 0.20.

End screen stats
----------------
- Battle ends when bossHp <= 0 or playerHp <= 0.
- Retreat also ends the battle as loss.
- The player is not auto-returned to the homescreen.
- End overlay shows:
  - Result title.
  - Result text.
  - Home button.
  - Damage.
  - Accuracy.
  - Time.
  - Panels disabled.
  - Charge hits.
  - Rating.
- Rating:
  - S: win, accuracy >= 70%, damageTaken <= 2, duration <= 45 seconds.
  - A: win, accuracy >= 50%.
  - B: any other win.
  - C: loss with damageDealt >= half of that boss's max HP.
  - D: any other loss.
